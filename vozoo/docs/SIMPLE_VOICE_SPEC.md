# かんたんボイス 仕様書（こども向けUI） v2

> ターゲット: **小学生〜高校生**。専門用語ゼロ・大きなボタン・**漢字にはふりがな**・即フィードバックを原則とする。
> 既存の Vozoo（録音 → エフェクト選択 → 再生/共有）に、子どもでも迷わず声を変えられる導線を追加する。
>
> v2: 設計レビュー反映。安全機構（リミッター/クランプ）はバックエンド新規実装の**要件**として確定。`ring_mod` mix・リバーブ統一・測定可能な受け入れ条件を反映。

## 1. ねらい / コンセプト

- 「録音したじぶんの声が、ボタンひとつで別のキャラの声になる」体験を最短ステップで届ける。
- 参考にしたゲームの声メーカー画面（`かんたん / カスタム` タブ + 大きなスライダー + キャラ選択）を踏襲。
- DSP（エフェクト）が得意なこと＝**声の高さ・太さ・ひびき**に機能を絞り、無理なこと（しゃべる速さ・抑揚・別人化）はやらない。これにより「いつでも自然に・安全に」動く。

## 2. ターゲットと配慮（アクセシビリティ）

| 観点 | 方針 |
|---|---|
| 文字 | **漢字にはふりがな**を付ける（全ひらがな固定にはしない）。本文18pt以上、見出し24pt以上。中高生にも違和感のない表記。 |
| 用語 | 「ピッチ」「フォルマント」等は使わない →「こえの たかさ」「こえの ふとさ」。 |
| 操作 | タップ領域は48dp以上。スライダーつまみは大きく、はじき防止。 |
| 色 | 高コントラスト。色だけに頼らずアイコン+絵文字も併用（色覚配慮）。 |
| 失敗しない | 値は**バックエンドで**安全範囲にクランプ。音量は**バックエンドで必ずリミッター**を通す（§6）。 |
| フィードバック | §8 の性能目標に従う。処理中はぐるぐる表示。完了で自動再生。 |

## 3. 画面フロー

```
[ホーム/録音] ──録音おわり──▶ [こえをへんしんしよう] ──OK──▶ [できた!(結果)]
                                   │  (かんたん / カスタム タブ)        │
                                   ◀───────── もう1かい ───────────────┘
```

既存 `HomeScreen`（録音）と `ResultScreen`（再生/共有）はそのまま活用。
**新規追加**するのは中央の「こえをへんしんしよう」画面（= 既存 `EffectSelectScreen` を子ども向けに置き換え/前面化）。
従来の `ChainEditorScreen` / `GraphEditorScreen`（上級者向け）は残すが、トップ導線からは「もっと」奥に隠す。

## 4. 画面仕様: こえをへんしんしよう

上部に2タブ（ゲームと同じ）。

### 4.1 「かんたん」タブ（既定で表示）

- 大きなキャラボタンを **2列×3 = 6個** グリッド表示（しゃべり方 1〜6 に相当）。
- 各ボタン = 絵文字 + 名前（漢字+ふりがな or ひらがな）+ 背景色。タップで選択し、即プレビュー加工（§8の連打防止に従う）。

各キャラは下記の**実在ノードのみ**で構成する。pitch は `semitones` 規約に統一。リバーブは `convolution_reverb` に統一（`dry_wet` で強度制御）。フォルマントは `formant_shift` を明示的に使う。

| # | なまえ | 絵文字 | チェーン（実ノード・上から順に適用） |
|---|---|---|---|
| 1 | ゴリラ | 🦍 | `pitch_shift{semitones:-5}` → `formant_shift{shift_factor:0.8}`（太い） |
| 2 | ねこ | 🐱 | `pitch_shift{semitones:+6}` → `formant_shift{shift_factor:1.25}`（細い） |
| 3 | ロボット | 🤖 | `ring_mod{mod_freq:50, quantize_steps:8, mix:0.8}` → `convolution_reverb{dry_wet:0.15}` |
| 4 | おばけ | 👻 | `pitch_shift{semitones:-2}` → `convolution_reverb{dry_wet:0.5}`（強いひびき） |
| 5 | ようせい | 🧚 | `pitch_shift{semitones:+9}` → `formant_shift{shift_factor:1.3}` → `chorus{mix:0.4}` → `convolution_reverb{dry_wet:0.2}`（キラキラ=高音+chorus+軽ひびき） |
| 6 | うちゅうじん | 👽 | `pitch_shift{semitones:-3}` → `ring_mod{mod_freq:80, quantize_steps:6, mix:0.6}` → `convolution_reverb{dry_wet:0.3}` |

- 末尾のリミッターは**書かない**（§6によりバックエンドが自動付与）。
- 選択中のキャラは枠を強調。下部に大きな「▶ きいてみる」と「OK」ボタン。

### 4.2 「カスタム」タブ

大きなスライダー（ゲーム画面踏襲）。**4本に限定**（多すぎない）。

| ラベル(表示) | 内部ノード.パラメータ | 範囲(UI) | 既定 | 備考 |
|---|---|---|---|---|
| 🎵 こえの たかさ | `pitch_shift.semitones` | -12 〜 +12 | 0 | 中央=そのまま（0で素通し） |
| 🫧 こえの ふとさ | `formant_shift.shift_factor` | 0.7 〜 1.4 | 1.0 | 左=太い/右=細い（1.0で素通し） |
| 🏞 ひびき | `convolution_reverb.dry_wet` | 0 〜 0.6 | 0 | 0=なし |
| 🤖 ロボットみ | `ring_mod.mix` | 0 〜 1 | 0 | **0=素の声に戻る**（要 §9 backend: mix追加） |

- 既定値（中央）のときは、そのノードを**チェーンに入れない**（= 素通し）。これで「0/中央 = 何もしない」を保証し、不要な処理も避ける。
  - たかさ=0、ふとさ=1.0、ひびき=0、ロボットみ=0 のノードは生成しない。
- スライダーは「中央スナップ」: つまみが既定値の **±5%** 内にあるとき既定値に吸着。±5%を超えて動かすと解除（低学年が真ん中に戻しやすく、高学年は微調整可）。
- 値は数字を見せず、つまみ位置だけ。左右端に小さな絵文字ヒント（例: たかさ→🐘/🐭）。
- 下部に「▶ きいてみる」「OK」。

> 注: 「話す速さ」「抑揚（上げ下げ）」はDSP単体では自然に作れないため、本バージョンでは**入れない**（§7参照）。

## 5. 結果画面（既存ResultScreen拡張）

- 再生バー + 大きな ▶/⏸。
- ボタン: 「💾 ほぞん」「📤 ともだちにおくる(共有)」「🔁 もう1かい(やり直し)」。
- 「もう1かい」で変身画面に戻る。
- 共有はOS共有シート経由（§6-3）。共有ボタン押下時に「しらない人に自分の声をおくらないでね」の注意文を1行表示。

## 6. 安全・データ仕様（重要・実装要件）

> 以下 1, 2 は現状の `process_file_with_chain`（`lib.rs:63`）／`build_node`（`chain_def.rs`）には**未実装**。本バージョンで**新規実装すること**を要件とする。

1. **音量保護（リミッター自動付与）**: `process_file_with_chain` 内で、ビルドしたチェーンの**末尾に必ずリミッターを付与**する（フロントが積むかどうかに依存しない）。これにより、どんなキャラ/スライダー組合せでも出力ピークが上限を超えない。
2. **値クランプ**: `build_node` が各ノードのパラメータを `available_nodes()` の min/max に従って安全範囲へクランプする（不正値・範囲外でも破綻しない）。
3. **プライバシー**: 録音・加工はすべて端末内（オフライン）で完結し、アプリ自身は音声をサーバへ送信しない。「共有」はOS共有シートを介したユーザー操作時のみ端末外へ出る。
4. **権限**: マイク権限のみ。許諾文言（案）:「こえを ろくおんするために マイクを つかいます。」

## 7. やること / やらないこと（スコープ）

**やる（このDSPベースで自然にできる）**
- 声の高さ・太さ・ひびき・ロボット声の変更
- 6キャラのワンタップ変身
- 録音 → 変身 → 再生/保存/共有

**やらない（DSP単体では不自然/別技術が必要）**
- 話す速さだけを変える：`pitch_shift_resample` は存在するが速さとピッチが連動するため「速さだけ」は不可。ピッチ非依存のタイムストレッチノードは未実装。
- 抑揚（上げ下げ）の作り替え：プロソディ編集が必要でDSP単体では不可。
- 完全な別人・別キャラへの自然な変換：AI音声変換/クローンが必要。

将来: 上記は AI（音声変換/クローン）で別フェーズとして検討（本仕様の範囲外）。

## 8. 受け入れ条件（Acceptance, 測定可能）

- [ ] 録音停止後、キャラを1タップすると自動でプレビュー再生される（停止タップを除き実質1タップ）。
- [ ] 「かんたん」タブの6キャラがすべて加工・再生できる。
- [ ] 「カスタム」タブの4スライダーが反映され、すべて既定値（中央）のとき素の声（素通し）になる。
- [ ] **音量保護テスト（自動）**: 代表的な極端チェーン（全スライダー最大、各キャラ）でエンジン処理した出力の**ピーク絶対値が 0.9（≒ -1 dBFS）以下**であること。`chain_def.rs` の既存テスト様式（出力サンプルの max を検証）で単体テスト化する。
- [ ] **クランプテスト（自動）**: 範囲外パラメータ（例 `gain.factor:100`）を渡しても、`available_nodes()` の max にクランプされること。
- [ ] **性能目標**: 10秒録音を実機で概ね2秒以内に加工完了（連打時は前処理をキャンセル/無視し、最後の選択のみ反映）。
- [ ] 表示テキストの漢字にはふりがなが付く（または平仮名）。
- [ ] オフラインで完結する（共有操作を除く）。

## 9. 実装メモ（技術対応）

**フロント**
- 新 `SimpleVoiceScreen`（タブ + キャラグリッド + スライダー）。`EffectChain`/`EffectNode` を組み立て `processWithChain` を呼ぶ。
- 既定値ノードは生成しない（§4.2）。連打時は最後の選択のみ処理（処理中フラグ + 最新値の再処理）。

**バックエンド（`vozoo-nodes`）— 新規実装が必要**
1. `chain_def.rs` `build_node`: 各ノードのパラメータを `available_nodes()` の min/max でクランプ。
2. `lib.rs` `process_file_with_chain`: ビルド後のチェーン末尾に**リミッターを自動付与**（既に末尾がリミッター系なら二重付与しない）。リアルタイム経路（`engine_set_chain`）にも同様の保護を検討。
3. `ring_mod.rs`: dry/wet **`mix`** パラメータを追加（0=dry/素通し, 1=full wet）。`chain_def.rs` の `ring_mod` ビルドと `available_nodes()`（min0,max1,default1）に反映。
4. キャラ用プリセットチェーンを `preset_chain_defs`（または新規 simple-voice プリセット定義）に §4.1 の内容で整理。pitch は `semitones` 規約に統一、リバーブは `convolution_reverb` に統一。
5. （リアルタイム経路向け・参考）`Reverb::reset`（`reverb.rs:49`）等の状態クリア漏れ。**オフライン加工（本仕様の主経路）は毎回チェーンを再ビルドするため実害なし**。リアルタイムモニタを使う場合のみ要対応。
